# 《她途》产品需求文档 PRD
**版本：** v1.0 初版  
**项目类型：** 黑客松 · 女性向网页互动叙事游戏  
**核心立意：** 人生没有对错之分的选择，只有做了选择之后，你怎么把它走成自己的路

---

## 一、产品概述

### 1.1 项目背景
《她途》是一款面向女性用户的网页端互动叙事小游戏，以"女性人生剧本"为切入点，通过 AI 驱动的个性化内容，让用户在模拟人生关键节点的选择中感受到：**每一条路都可以走出光来，区别只在于你怎么走**。

### 1.2 核心传递的价值观
- 选择本身没有对错好坏之分
- 人生不像游戏可以存档回退，25岁过了就不会再有25岁
- 珍惜当下，把握此刻能改变的一切
- 打破"女性人生只有一种标准剧本"的单一叙事

### 1.3 目标用户
主要面向 18–35 岁的中国女性用户，尤其是正在经历或即将面临社会压力节点（升学、就业、婚恋、职场）的群体。

---

## 二、完整用户流程

```
封面页
  ↓ 点击「开始她的故事」
信息填写页（昵称 + 出生年份）
  ↓ 点击「开始旅程」
节点 1：17岁 · 志愿的岔路口
  ↓ 选择其中一项 → AI 生成"选择后的风景" → 点击继续
节点 2：22岁 · 毕业季的风
  ↓ 同上
节点 3：25岁 · 催婚的那顿饭
  ↓ 同上
节点 4：28岁 · 事业的十字路
  ↓ 同上
节点 5：32岁 · 此刻的你
  ↓ 同上 → 进入平行宇宙环节
与「平行的她」对话（AI 角色扮演）
  ↓ 用户主动点击「回到自己的故事」
结尾页（AI 个性化信件 + 所有选择回顾）
  ↓ 点击「生成分享长图」
分享卡展示（截图保存 + 二维码）
```

## 三、页面与功能详细说明

### 3.1 封面页

**视觉要素：**
- 游戏标题：**她途**（大字，像素风衬线字体）
- 副标题：`HER WAY · HER WORLD`
- 引导语：「人生没有存档，没有读取，没有回退。但每一个当下，都是你能重新开始的地方。」
- 像素装饰元素（横向像素点阵）
- 版权信息：`© 她途 · FOR EVERY WOMAN WHO CHOSE HER OWN WAY`

**交互：**
- 唯一按钮：`▸ 开始她的故事`（闪烁动效，16bit 风格）
- 点击后进入信息填写页

---

### 3.2 信息填写页

**收集字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 昵称 | 文本输入 | 最多10字，用于全程个性化称呼 |
| 出生年份 | 数字输入 | 范围 1985–2008，用于结尾信件的个性化表达 |

**设计说明：**
- 出生年份**仅在结尾信件中使用**，计算用户与各节点年龄的关系（已经经历过 / 还没到 / 正处于）
- 所有用户无论实际年龄，都完整走完 17→22→25→28→32 岁的完整五节点流程
- 填写完成后显示个性化欢迎语，例如：`小鱼，你的故事从这里开始。`

**校验规则：**
- 昵称不可为空
- 出生年份须为 1985–2008 之间的整数，否则提示错误

---

### 3.3 人生节点页（共 5 个，结构相同）

#### 节点结构

每个节点包含以下固定结构：

1. **顶部进度条**：5格，当前节点高亮金黄色，已完成节点为樱花粉，未完成为深色
2. **年龄标签**：`XX岁 · 节点标题`（金黄色像素标签）
3. **情景描述**：融入用户昵称的叙事文本，约 60–80 字
4. **3个选项卡片**：每张卡片包含像素插画 + 主标题 + 副标题
5. **选择后展示**：选中某项后，其余选项消失，出现 AI 生成的"选择后的风景"文字（约 80 字）
6. **继续按钮**：`▸ 继续前行`（最后一个节点显示 `▸ 看看她的故事`）

#### 5个节点内容

**节点 1 · 17岁 · 志愿的岔路口**

情景：高考成绩刚出来，你坐在饭桌前，爸妈递过来一张填好的志愿表——师范或医学，稳定，体面，离家近。你握着笔，窗外的风吹动了窗帘。

| 选项 | 主标题 | 副标题 |
|------|--------|--------|
| A | 填了他们选的专业 | 稳定，是一种爱 |
| B | 坚持填了自己的选择 | 那一刻，笔很重 |
| C | 随便填了一个，走一步看一步 | 反正路还长着呢 |

---

**节点 2 · 22岁 · 毕业季的风**

情景：毕业典礼结束了，同学们抱着哭，有人要回老家，有人要去北上广，有人要出国。你站在操场上，感受着最后一次校园的风，想到那个关于未来的问题还没有答案。

| 选项 | 主标题 | 副标题 |
|------|--------|--------|
| A | 回老家，备考公务员 | 安稳，是另一种勇气 |
| B | 去大城市，从头开始 | 一个人，一个行李箱 |
| C | 出国读研，再看看世界 | 还不想停下来 |

---

**节点 3 · 25岁 · 催婚的那顿饭**

情景：春节饭桌上，七大姑八大姨的眼神都落在你身上。「都25了，有对象了吗？」你夹起一块鱼，慢慢放进嘴里，想好了要怎么回答。

| 选项 | 主标题 | 副标题 |
|------|--------|--------|
| A | 顺势相亲，认真试试 | 也许缘分就在这里 |
| B | 礼貌拒绝，专注自己 | 我的时钟，我来定 |
| C | 谈着恋爱，但不着急结果 | 感情的事，急不来 |

---

**节点 4 · 28岁 · 事业的十字路**

情景：你的上司找你谈话，有一个升职机会——更高的薪水，但要牺牲大部分私人时间。同一周，你心里另一个声音也越来越响：那件一直想做但没做的事。

| 选项 | 主标题 | 副标题 |
|------|--------|--------|
| A | 接受升职，全力以赴 | 拼过这几年，以后再说 |
| B | 辞职，去做那件想做的事 | 现在不做，以后更难 |
| C | 留下来，但划定了自己的边界 | 工作是工作，我还是我 |

---

**节点 5 · 32岁 · 此刻的你**

情景：你坐在某个地方——可能是家里，可能是咖啡馆，可能是一趟还没到站的列车上。回头看那些年，那些选择，那些没走的路。你想对此刻的自己说什么？

| 选项 | 主标题 | 副标题 |
|------|--------|--------|
| A | 我已经找到了答案 | 不管哪条路，都是我的路 |
| B | 我还在寻找中 | 没关系，寻找本身就是意义 |
| C | 答案本身不重要 | 活着，就是在写答案 |

---

#### AI 生成：选择后的风景

每次用户做出选择后，调用 Claude API 生成约 80 字的叙述，要求：
- 描述的是"这个选择之后她看到的风景"，而非结果评判
- 语气温柔、诗意，无对错暗示
- 融入用户昵称
- Prompt 模板：
```
玩家叫"[昵称]"，在[年龄]，面对"[节点标题]"，她选择了：「[选项主标题]」（[副标题]）。
请用80字左右，描述她在这个选择之后看到的"风景"——不是结果，而是感受和画面。
不要有评判，不要说这个选择好或不好。要有诗意，但真实。
直接输出内容，不要任何前缀。
```

---

### 3.4 与「平行的她」对话页

**触发时机：** 5个节点全部完成后进入

**功能说明：**
- 系统根据用户的5个选择，生成一个"做了完全相反选择的平行自己"
- 平行的她自动发出一段自我介绍（约 60 字）
- 用户可以自由和她多轮对话
- 用户主动点击「回到自己的故事」按钮后进入结尾页

**平行角色设定原则：**
- 用第一人称，平静、真实，有时有遗憾，有时庆幸，但不后悔自己的路
- 不比较谁的选择更好，不评判
- 每次回复不超过 80 字

**Prompt 模板（System）：**
```
你是"[昵称]"在平行宇宙里的另一个版本。
原本的她的选择：[各节点选择汇总]
你（平行的她）的选择：[各节点相反选择汇总]

用第一人称和她对话。语气真实、平静，有时会有遗憾，有时会庆幸，但不后悔自己的路。
每次回复不超过80字。不要说教，不要比较谁更好。
```

---

### 3.5 结尾页

**内容构成：**

1. **AI 个性化信件**（约 100 字）
   - 开头：`[昵称]，`
   - 内容融入出生年份：计算用户实际年龄与节点年龄的关系，给出有代入感的一句话（例：「你今年26岁，32岁对你来说还有6年」或「也许你已经走过了这些岁月」）
   - 逐一提及用户的5个选择，但不评判好坏
   - 结尾固定落点：**此刻，就是你能改变的起点**
   - 信件最后一行固定：`人生没有存档，但每一步都算数。`

2. **选择回顾列表**
   - 展示5个节点的年龄标签 + 用户所选选项主标题 + 对应像素插画

3. **生成分享长图按钮**

---

### 3.6 分享卡

**展示方式：** 点击按钮后在页面内展开，用户截图保存

**卡片内容（从上到下）：**

```
┌──────────────────────────────┐
│  ▸ 她途 · HER WAY            │  （顶部标识，8px像素字）
│                              │
│  [昵称] 的故事               │  （22px，强调字）
│  ━━━━━━                      │  （粉色分割线）
│                              │
│  · 在17岁，[选项A主标题]     │
│  · 在22岁，[选项B主标题]     │  （5条选择记录）
│  · 在25岁，[选项C主标题]     │
│  · 在28岁，[选项A主标题]     │
│  · 在32岁，[选项B主标题]     │
│                              │
│  [AI生成的个性化一句话]       │  （约30字，信件精华提炼）
│                              │
│  人生没有存档，               │
│  但每一步都算数。             │  （固定 Slogan）
│                              │
│  [二维码]  扫码开始你的她途   │  （二维码 + 引导语）
└──────────────────────────────┘
```

**二维码：** 指向本游戏网址，使用 `qrcode.react` 组件生成

**保存方式：** 提示用户截图保存（初版不做服务端图片生成）

## 四、AI 调用汇总

| 调用时机 | 输入 | 输出 | 字数要求 |
|---------|------|------|---------|
| 每个节点选择后 | 昵称 + 年龄 + 节点 + 选项 | 选择后的风景描述 | 约 80 字 |
| 平行宇宙介绍 | 昵称 + 5个正向选择 + 5个反向选择 | 平行的她的自我介绍 | 约 60 字 |
| 平行宇宙对话 | 全部上下文 + 用户输入 | 平行的她的回复 | 每次 ≤ 80 字 |
| 结尾信件 | 昵称 + 出生年份 + 5个选择 | 完整个性化信件 | 约 100 字 |
| 分享卡一句话 | 信件内容 | 精华提炼一句话 | ≤ 30 字 |

**模型：** `claude-sonnet-4-6`  
**max_tokens：** 1000  
**无需后端**，全部在前端直接调用 Anthropic API

## 五、视觉设计规范

### 5.1 色彩系统

| 色彩用途 | 色值 | 命名 |
|---------|------|------|
| 背景主色 | `#0D0D1A` | 深夜蓝黑 |
| 卡片背景 | `#0A0A14` | 深黑 |
| 次级背景 | `#1A1A2E` | 深蓝黑 |
| 正文文字 | `#F0E6D3` | 暖米白 |
| 次级文字 | `#C4B8D8` | 淡紫灰 |
| 辅助文字 | `#8B7EA0` | 中紫灰 |
| 禁用/装饰 | `#4A3F6B` | 深紫 |
| 强调色 · 粉 | `#E8829A` | 樱花粉 |
| 强调色 · 金 | `#F5C842` | 像素金 |
| 强调色 · 紫 | `#9B59B6` | 平行紫（专用于平行宇宙环节） |

### 5.2 字体

| 用途 | 字体 | 说明 |
|------|------|------|
| 标签 / 按钮 / 像素装饰 | `Press Start 2P` | Google Fonts，16bit 像素风 |
| 正文 / 标题 / 叙述文字 | `ZCOOL XiaoWei` | Google Fonts，中文衬线，有东方气质 |

### 5.3 像素插画规范

- 每个选项配 1 张 64×64px 的像素插画
- 技术方案：纯 SVG + `<rect>` 手工绘制，`imageRendering: pixelated`
- 共 15 张（5节点 × 3选项），每张均有独特场景与色调
- 风格：16bit 复古游戏风，有女性角色 / 场景元素

### 5.4 交互细节

- **像素边框**：所有卡片、按钮、输入框使用 `box-shadow` 模拟4方向像素边框（非 CSS border）
- **扫描线叠加**：全屏固定定位的扫描线 overlay，增强复古屏幕感
- **按钮闪烁**：封面主按钮 600ms 间隔闪烁（文字颜色切换）
- **hover 效果**：选项卡片 hover 时像素边框变为樱花粉
- **进度条**：顶部 5 格像素进度条，随节点推进实时更新
- **过渡动画**：页面切换淡入淡出（Framer Motion）

## 六、技术架构

### 6.1 技术栈

| 模块 | 方案 |
|------|------|
| 框架 | React（Vite） |
| 样式 | 纯 inline style（无 CSS 框架依赖） |
| 动效 | Framer Motion |
| AI | Claude API（`claude-sonnet-4-6`） |
| 二维码 | `qrcode.react` |
| 分享图 | 截图方案（初版）/ html2canvas（迭代版） |
| 数据存储 | 纯前端 React state，无后端，无数据库 |

### 6.2 页面状态机

```
cover → setup → node[0] → node[1] → node[2] → node[3] → node[4] → parallel → ending
```

State 中维护：
- `playerName`：昵称
- `birthYear`：出生年份
- `nodeIndex`：当前节点索引（0–4）
- `choices`：用户5次选择的记录数组 `[{ key, opt }]`
- `screen`：当前屏幕状态

### 6.3 API Key 配置

在 `.env` 文件中配置：
```
VITE_ANTHROPIC_API_KEY=your_key_here
```

fetch 时从环境变量读取，不硬编码在源码中。
